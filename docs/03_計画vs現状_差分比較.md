## 2. ナレッジ・プロンプト設計の差分

### 2.1 計画（理想の設計）

```
ナレッジ3層構造:
  ├── 共通ナレッジ (栄養・ダイエット基礎知識)
  │     → 全ユーザー共通で参照
  ├── 運用ナレッジ (クリニック固有のFAQ・指導方針)
  │     → admin単位で個別設定
  └── 個人ナレッジ (ユーザーの問診結果・記録履歴)
       → ユーザー単位で自動蓄積

プロンプト構成:
  システムプロンプト = 基本人格 + 共通ナレッジ + 運用ナレッジ + 個人コンテキスト
```

### 2.2 現状の実装

| 層 | DB テーブル | 実装状況 |
|----|-----------|----------|
| **BOT定義** | `bots` → `bot_versions` | ✅ テーブル存在, レコードあり (diet-support-bot-v1), **❌ コードから未参照** |
| **ナレッジベース** | `knowledge_bases` | ✅ テーブル存在, 2件登録 (基礎知識/クリニックFAQ), **❌ コードから未参照** |
| **ナレッジ文書** | `knowledge_documents` | ✅ テーブル存在, **❌ 0件 (空)** |
| **BOT↔ナレッジリンク** | `bot_knowledge_links` | ✅ テーブル存在, 2件リンク済み, **❌ コードから未参照** |
| **相談プロンプト** | (ハードコード) | ✅ process-line-event.ts 内に直書き |
| **画像解析プロンプト** | (ハードコード) | ✅ image-analysis.ts / image-analyzer.ts に直書き |

### 2.3 不足している実装

| # | 不足点 | 優先度 | 影響 | 推奨対応 |
|---|--------|--------|------|----------|
| 1 | **ナレッジ文書が空** | 🔴高 | 共通ナレッジ（基礎知識）が相談に反映されない | knowledge_documents にコンテンツ投入 |
| 2 | **BOTバージョンのプロンプトが未使用** | 🔴高 | bot_versions.system_prompt を使わず、ハードコードプロンプトを使用 | handleConsultText でDB上のプロンプトを参照するよう改修 |
| 3 | **ナレッジ検索(RAG)未実装** | 🟡中 | knowledge_documentsのembedding検索がない | createEmbedding()は実装済みだが、検索フローが未接続 |
| 4 | **個人コンテキスト未注入** | 🟡中 | 相談時にユーザーの問診結果・過去記録を参照しない | systemPromptにuser_profiles/recent_logsを動的追加 |
| 5 | **プロンプト管理画面なし** | 🟢低 | プロンプト変更にはコード変更が必要 | 管理画面にプロンプトエディタ追加（Phase 2） |
