## 2. 今回のバグ修正（v1.0.0 → v1.1.0）

### 2.1 修正済みバグ

| # | バグ | 原因 | 影響 | 修正内容 |
|---|------|------|------|----------|
| **BUG-1** | **replyTextWithQuickReplies 未インポート** | process-line-event.ts で使用しているが import に含まれていなかった | 記録モード・相談モードで返信時に**ランタイムクラッシュ** (ReferenceError) | `import { replyTextWithQuickReplies }` を追加 |
| **BUG-2** | **handleFollowEvent で ensureUserAccount 二重呼出し** | ステップ3で呼んだ後、ステップ5で再度呼んでいた | 不要な DB 書き込み、タイミングによっては UNIQUE 制約エラー | ステップ5で findUserAccount のみに変更（既存チェック） |
| **BUG-3** | **ensureUserAccount の SELECT+INSERT 競合** | findUserAccount → createUserAccount の間に並行リクエストが入るとUNIQUE制約違反 | 「既に使用中」エラーの根本原因 | `INSERT ... ON CONFLICT DO UPDATE` に書き換え（atomic upsert） |
| **BUG-4** | **招待コード同一アカウント再紐付け** | 同一ユーザーが異なるコードで同じ account_id に紐付け可能だった | 不要な usage レコード増加 | useInviteCode で同一account_id への重複チェック追加 |

### 2.2 BUG-1 の詳細分析（最も深刻）

`replyTextWithQuickReplies` は `reply.ts` で export されていたが、`process-line-event.ts` では **import されていなかった**。これにより：

1. ユーザーがテキストメッセージ（体重や食事テキスト）を送信
2. 記録処理は成功（DB書き込み完了）
3. 返信時に `replyTextWithQuickReplies is not defined` で **クラッシュ**
4. LINE に返信が届かないため、ユーザーが「登録できていない」と誤解
5. LINE からの再送 or ユーザーの再操作で、follow イベントの再処理が発生
6. 既に存在する user_accounts に再 INSERT → **UNIQUE 制約エラー**

この連鎖が「新規IDが既に使用中」エラーの主因。

## 3. ナレッジ・プロンプト設計の差分（更新後）

### 3.1 実装完了した3層構造

```
ナレッジ3層構造:
  ├── 共通ナレッジ (栄養・ダイエット基礎知識)     ← ✅ 8件投入済み
  │     → knowledge_documents (kb_common)
  ├── 運用ナレッジ (クリニック固有のFAQ・指導方針) ← 🔶 テーブル/リンク存在、コンテンツ投入は要運用
  │     → knowledge_documents (kb_client)
  └── 個人ナレッジ (ユーザーの問診結果・記録履歴)   ← ✅ buildUserContext で動的構築
       → user_profiles + body_metrics + meal_entries

プロンプト構成:
  システムプロンプト = DB上のbot_versions.system_prompt  ← ✅ 実装済み
                     + 個人コンテキスト (buildUserContext)  ← ✅ 実装済み
                     + ナレッジ検索結果 (searchKnowledgeForBot)  ← ✅ 実装済み
```

### 3.2 ナレッジDB接続のフロー

```
ユーザー「〜について教えて」
  ↓
1. getPublishedSystemPrompt(accountId)
   → bot_versions.system_prompt から基本プロンプト取得
   → なければハードコードフォールバック
  ↓
2. buildUserContext(userAccountId)
   → user_profiles (プロフィール情報)
   → body_metrics (直近7日の体重推移)
   → meal_entries (今日の食事記録)
  ↓
3. searchKnowledgeForBot(accountId, query)
   → キーワード抽出 → LIKE検索
   → bot_knowledge_links経由で紐付くドキュメントを返す
  ↓
4. systemPrompt = 基本プロンプト + 個人コンテキスト + ナレッジ
  ↓
5. OpenAI Chat Completions → LINE返信
```
