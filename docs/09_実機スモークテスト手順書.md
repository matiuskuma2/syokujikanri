# 09 実機スモークテスト手順書

## 概要

本番デプロイ後に LINE アプリ実機で行うスモークテスト。  
E2E テスト（79 件 PASS）で検証済みのフローを、実際の LINE UI で確認する。

| 項目 | 値 |
|------|-----|

| LINE Webhook | `https://diet-bot.pages.dev/api/webhooks/line` |
| LIFF URL | `https://liff.line.me/2009409790-DekZRh4t` |
| テスト用招待コード | 本番 DB に事前発行（管理画面から作成） |
| Rich Menu | 3 ボタン: 記録する / 相談する / ダッシュボード |

## テスト準備

### 1. LINE 公式アカウント設定確認

- [ ] Webhook URL が `https://diet-bot.pages.dev/api/webhooks/line` に設定されている
- [ ] Webhook 検証（Verify）が成功する
- [ ] 応答メッセージが「無効」に設定されている（BOT が二重応答しないように）
- [ ] あいさつメッセージが「無効」に設定されている

### 2. テスト用招待コードの発行

管理画面（`/admin/invite-codes`）でテスト用コードを発行：
- コード例: `TST-9901`（max_uses: 2 以上にして複数テスト可能に）
- またはスーパー管理画面から直接 SQL で投入

### 3. テストアカウント

- テスト用 LINE アカウント（本番ユーザーと混ざらないよう注意）
- 友だち追加前の状態から開始

## テスト手順

### ST-1: 新規ユーザー → 招待コード送信

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 1-1 | LINE 公式アカウントを友だち追加 | 「友だち追加ありがとう！招待コードを送信してください」メッセージが届く |
| 1-2 | 招待コード（例: `TST-9901`）を送信 | 「登録完了！」+「質問 1/9: ニックネームは？」が届く |

**確認ポイント:**
- [ ] フォローイベントで友だち追加メッセージが来る
- [ ] 招待コード送信で登録完了メッセージ + Q1 が来る

---

### ST-2: 問診を約 3 問進める

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 2-1 | ニックネーム（例: `テストユーザー`）を送信 | 「テストユーザーさん、よろしく！ Q2: 性別は？」 |

**確認ポイント:**

## トラブルシューティング

### Webhook が応答しない
1. `curl https://diet-bot.pages.dev/api/health` で 200 が返るか確認
2. LINE Developers Console で Webhook URL を再確認
3. Webhook 検証ボタンで通信テスト

### AI 応答がない（相談モード）
1. 本番の `OPENAI_API_KEY` が設定されているか確認  
   `npx wrangler pages secret list --project-name diet-bot`
2. OpenAI API の残高・レートリミットを確認

### 画像解析が動かない
1. R2 バケット (`diet-bot-media`) が存在するか確認
2. `OPENAI_API_KEY` に GPT-4 Vision 権限があるか確認

### LIFF が開かない
1. LINE Developers Console で LIFF アプリの Endpoint URL を確認
2. `https://liff.line.me/2009409790-DekZRh4t` にブラウザでアクセスしてみる
